# Ubiquitous Language

## Infrastructure provisioning

| Term | Definition | Aliases to avoid |
|------|-----------|-----------------|
| **Tofu module** | A self-contained OpenTofu directory that provisions a specific set of cloud resources | Terraform module, tofu dir |
| **Provider** | A cloud platform target for infrastructure provisioning (aws, gcp, harvester) | Cloud, backend |
| **Workspace** | An isolated OpenTofu state namespace within a module | Environment (when referring to Tofu isolation) |
| **tfvars** | A file of variable assignments passed to a Tofu module at apply time | vars file, config file |
| **Tofu output** | A named value exported from a Tofu module after apply, consumed by downstream tools | Terraform output |

## Environments

| Term | Definition | Aliases to avoid |
|------|-----------|-----------------|
| **ENV** | The deployment topology type; one of `default`, `airgap`, or `proxy` | environment type, deployment type |
| **default** | An internet-connected deployment where nodes have public IPs | standard, normal |
| **airgap** | A network-isolated deployment where nodes have no direct internet access | air-gapped, offline |
| **proxy** | A deployment where internet access is routed through an HTTP proxy | proxied |
| **Bastion host** | The internet-facing jump host used to reach airgap nodes | jump host, bastion server |

## Cluster nodes

| Term | Definition | Aliases to avoid |
|------|-----------|-----------------|
| **Node** | A single compute instance that participates in a Kubernetes cluster | Server (in the generic sense), machine, VM |
| **Node role** | A Tofu-layer label describing a node's cluster function: `cp`, `etcd`, or `worker` | node_roles, role (when referring to Tofu roles) |
| **Distro** | The Kubernetes distribution being deployed: `rke2` or `k3s` | flavor, variant |

## RKE2 node classification

| Term | Definition | Aliases to avoid |
|------|-----------|-----------------|
| **master** | The first RKE2 node; initializes the cluster and holds the bootstrap token | primary, init node, first server |
| **server** | An RKE2 control-plane node that joins an existing cluster (not the first) | cp node, etcd node |
| **agent** | An RKE2 node that runs workloads only, with no control-plane function | worker (in the RKE2 context) |
| **rke2_node_role** | The Ansible-layer string (`master`, `server`, or `agent`) assigned per host in the inventory | node role (when referring to Ansible classification) |

## Ansible inventory

| Term | Definition | Aliases to avoid |
|------|-----------|-----------------|
| **Inventory** | The static YAML file that maps hostnames to IPs and host variables for a deployment | hosts file, inventory.yml |
| **Inventory schema** | The YAML file (`ansible/_inventory-schema.yaml`) that maps node roles to Ansible groups per distro/ENV | schema, mapping |
| **Inventory manifest** | The generated `.inventory-manifest.json` file containing checksums and metadata for a produced inventory | manifest, checksum file |
| **Group vars** | YAML variables applied to all hosts in an Ansible group | group variables, vars |
| **Role** | A reusable, self-contained Ansible unit of tasks (e.g. `rke2_config`, `rancher_auth`) | module (in the Ansible context) |

## Rancher

| Term | Definition | Aliases to avoid |
|------|-----------|-----------------|
| **Rancher** | The SUSE cluster management platform deployed on top of a Kubernetes cluster | RKE, the management layer |
| **API token** | A scoped Rancher credential generated by the `rancher_auth` role for downstream automation | access key, bearer token |

## Relationships

- A **Tofu module** produces one or more **Tofu outputs** consumed by the **Inventory** generator
- Each **Node** has one or more **node roles** (Tofu layer) and exactly one **rke2_node_role** (Ansible layer)
- An **Inventory** is generated per **Distro** + **ENV** combination
- Every **Inventory** is accompanied by exactly one **Inventory manifest**
- A **Bastion host** is present only in **airgap** deployments and acts as the SSH gateway to all **Nodes**

## Example dialogue

> **Dev:** "Should the `generate_inventory` script use the node's **node roles** to decide whether it's a **master**?"
>
> **Domain expert:** "Only partially — **node roles** tell you the Tofu-layer function (`cp`, `etcd`, `worker`). But **rke2_node_role** is the Ansible classification. The first node assigned to the `master` group gets `rke2_node_role: master`; any other node with `cp` or `etcd` gets `server`; pure worker nodes get `agent`."
>
> **Dev:** "So two nodes can both have the `cp` **node role** but have different **rke2_node_roles**?"
>
> **Domain expert:** "Exactly. The **Inventory schema** uses `first_only: true` on the `master` group to ensure only the first `cp`/`etcd` node becomes the **master**. All others are **servers**."
>
> **Dev:** "And in an **airgap** ENV, where does the **Inventory** come from?"
>
> **Domain expert:** "The airgap **Tofu module** emits a dedicated `airgap_inventory_json` **Tofu output** — the generator reads that instead of `cluster_nodes_json`. The **Bastion host** is included as its own group."

## Flagged ambiguities

- **"role"** is overloaded: it means an Ansible **Role** (a reusable task unit) AND a **node role** (a Tofu-layer label like `cp`). Use "Ansible role" and "node role" when the context is not obvious.
- **"worker"** appears as both a **node role** value (`worker`) in Tofu and as an Ansible group name in k3s inventories. In RKE2 context, the Ansible equivalent of a worker is an **agent**, not a worker. Prefer **agent** when referring to RKE2 Ansible classification.
- **"master"** is both an Ansible group name and the value of **rke2_node_role** for the first node. They refer to the same node, but the group is an Ansible inventory concept while `rke2_node_role: master` is a host variable. Distinguish by context.
- **"environment"** was used loosely to mean both **ENV** (the topology type: default/airgap/proxy) and the shell environment. Use **ENV** for the topology concept.
